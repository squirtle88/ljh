# 功能模块封装函数原型与调用关系

## 一、模块划分

根据系统业务需求，本系统可划分为七个功能模块：

- 数据结构基础模块：负责系统初始化、车位栈、等待队列、车牌哈希表等底层结构。
- 停车业务处理模块：负责车辆入场、出场、查找和排队调度。
- 计费模块：负责停车费用计算和计费规则设置。
- 统计分析模块：负责收入统计、楼层使用率、历史交易和高峰时段分析。
- 用户界面模块：负责菜单交互和功能入口。
- 数据持久化模块：负责数据保存、加载和索引重建。
- 主控模块：负责程序入口、初始化、菜单循环和退出清理。

## 二、数据结构基础模块

```c
void ds_init(ParkingSystem *ps);
```

初始化停车场系统。`ps` 为系统全局状态指针，不能为空。函数无返回值。执行后，60 个车位全部为空闲状态，空闲车位栈、等待队列、哈希表、历史记录数组、默认计费规则和累计收入均完成初始化。

```c
void ds_destroy(ParkingSystem *ps);
```

销毁停车场系统。`ps` 为系统全局状态指针，可为空。函数无返回值。主要释放空闲车位栈、等待队列、哈希表和历史交易数组等动态内存。

```c
void stack_init(FreeSpotStack *s, int capacity);
void stack_destroy(FreeSpotStack *s);
void stack_push(FreeSpotStack *s, SpotLocation loc);
SpotLocation stack_pop(FreeSpotStack *s);
int stack_is_empty(const FreeSpotStack *s);
int stack_size(const FreeSpotStack *s);
```

以上函数用于管理空闲车位栈。`s` 为栈指针，`capacity` 为初始容量，通常为 `TOTAL_SPOTS`；`loc` 为车位坐标，范围为合法楼层、行、列。`stack_pop` 返回一个合法车位坐标；`stack_is_empty` 返回非 0 表示栈空；`stack_size` 返回当前空闲车位数量，范围为 `0 ~ TOTAL_SPOTS`。

```c
void queue_init(WaitingQueue *q);
void queue_clear(WaitingQueue *q);
void queue_enqueue(WaitingQueue *q, const char *plate, time_t t);
int queue_dequeue(WaitingQueue *q, char *out_plate, time_t *out_t);
int queue_is_empty(const WaitingQueue *q);
int queue_count(const WaitingQueue *q);
```

以上函数用于管理等待队列。`q` 为队列指针；`plate` 为车牌号，长度范围为 `1 ~ 15`；`t` 为入队时间。`queue_dequeue` 成功出队返回 1，队列为空返回 0；`queue_is_empty` 返回非 0 表示队列为空；`queue_count` 返回等待车辆数量。

```c
unsigned int hash_str(const char *str);
void ht_init(HashMap *ht);
void ht_insert(HashMap *ht, VehicleRecord *rec);
VehicleRecord *ht_find(HashMap *ht, const char *plate);
void ht_remove(HashMap *ht, const char *plate);
void ht_clear(HashMap *ht);
```

以上函数用于管理车牌哈希表。`str` 和 `plate` 表示车牌字符串；`ht` 为哈希表指针；`rec` 为在场车辆记录。`hash_str` 返回 `0 ~ HT_SIZE-1` 的哈希下标；`ht_find` 找到车辆时返回 `VehicleRecord *`，未找到返回 `NULL`。

```c
SpotLocation make_loc(int f, int r, int c);
int loc_equal(SpotLocation a, SpotLocation b);
```

`make_loc` 用于创建车位坐标，`f`、`r`、`c` 分别表示楼层、行、列，取值范围分别为 `0 ~ FLOORS-1`、`0 ~ ROWS-1`、`0 ~ COLS-1`。`loc_equal` 判断两个坐标是否相同，相同返回非 0，否则返回 0。

## 三、停车业务处理模块

```c
EntryResult vehicle_entry(ParkingSystem *ps, const char *plate);
```

办理车辆入场。`ps` 为系统指针，不能为空；`plate` 为车牌号，长度范围为 `1 ~ 15`。返回 `ENTRY_SUCCESS` 表示入场成功，返回 `PARKING_FULL` 表示停车场已满并进入等待队列，返回 `PLATE_EXISTS` 表示车牌非法、重复或已排队。

```c
ExitResult vehicle_exit(ParkingSystem *ps, const char *plate, double *out_fee);
```

办理车辆出场结算。`ps` 为系统指针，不能为空；`plate` 为车牌号；`out_fee` 用于输出停车费用，可为空。返回 `PARK_EXIT_SUCCESS` 表示出场成功，返回 `NOT_FOUND` 表示未找到车辆，返回 `NOT_PARKED` 表示车辆正在排队但尚未停车。

```c
VehicleRecord *find_vehicle(ParkingSystem *ps, const char *plate);
int find_in_queue(ParkingSystem *ps, const char *plate);
```

`find_vehicle` 根据车牌查找在场车辆，找到返回车辆记录指针，未找到返回 `NULL`。`find_in_queue` 查询车辆在等待队列中的位置，找到返回从 1 开始的排队序号，未找到返回 `-1`。

## 四、计费模块

```c
double calculate_fee(const FeeRule *rule, int duration_minutes);
```

根据停车时长计算费用。`rule` 为计费规则指针，不能为空；`duration_minutes` 为停车时长，范围为 `>= 0`。返回值为停车费用，范围为 `>= 0`。

```c
void set_fee_rule(ParkingSystem *ps, FeeRule rule);
```

设置计费规则。`ps` 为系统指针，不能为空；`rule` 包含起步价、免费时长、小时费率和每日封顶金额。函数无返回值，执行后更新 `ps->fee_rule`。

## 五、统计分析模块

```c
double stats_total_revenue(const ParkingSystem *ps);
int stats_floor_used_count(const ParkingSystem *ps, int floor);
double stats_floor_usage_rate(const ParkingSystem *ps, int floor);
void stats_peak_hours(const ParkingSystem *ps, int hour_counts[24]);
void stats_print_transactions(const ParkingSystem *ps);
```

统计模块函数均以 `ParkingSystem *ps` 为主要参数。`stats_total_revenue` 返回累计收入；`stats_floor_used_count` 返回指定楼层已占用车位数，范围为 `0 ~ ROWS*COLS`；`stats_floor_usage_rate` 返回楼层使用率，范围为 `0.0 ~ 1.0`；`stats_peak_hours` 将 24 小时流量统计写入 `hour_counts` 数组；`stats_print_transactions` 输出历史交易明细。

## 六、用户界面模块

```c
void ui_main_menu(ParkingSystem *ps);
void ui_entry(ParkingSystem *ps);
void ui_exit(ParkingSystem *ps);
void ui_find(ParkingSystem *ps);
void ui_free_spots(ParkingSystem *ps);
void ui_wait_queue(ParkingSystem *ps);
void ui_parking_map(ParkingSystem *ps);
void ui_revenue(ParkingSystem *ps);
void ui_transactions(ParkingSystem *ps);
void ui_floor_usage(ParkingSystem *ps);
void ui_peak_hours(ParkingSystem *ps);
void ui_save(ParkingSystem *ps);
void ui_load(ParkingSystem *ps);
```

用户界面模块函数主要负责菜单显示、输入读取和结果输出。所有函数参数均为 `ParkingSystem *ps`，不能为空。UI 层不直接修改底层数据，而是调用业务层、统计层和持久化层接口完成操作。

## 七、数据持久化模块

```c
int save_data(ParkingSystem *ps, const char *filename);
int load_data(ParkingSystem *ps, const char *filename);
void rebuild_hash_from_spots(ParkingSystem *ps);
```

`save_data` 将停车场状态保存到文件，`load_data` 从文件恢复系统状态，`filename` 为非空文件名。二者成功返回 1，失败返回 0。`rebuild_hash_from_spots` 根据车位状态重建车牌哈希表，无返回值。

## 八、主控模块

```c
int main(void);
```

程序入口函数，无参数。主要流程为：创建 `ParkingSystem`，调用 `ds_init` 初始化系统，调用 `load_data` 尝试加载数据，进入 `ui_main_menu` 菜单循环，退出前调用 `save_data` 保存数据，最后调用 `ds_destroy` 释放资源。正常结束返回 0。

## 九、函数调用关系

```text
main
 ├─ ds_init
 ├─ load_data
 │   └─ rebuild_hash_from_spots
 ├─ ui_main_menu
 │   ├─ ui_entry
 │   │   └─ vehicle_entry
 │   │       ├─ ht_find
 │   │       ├─ find_in_queue
 │   │       ├─ stack_is_empty / stack_pop
 │   │       ├─ ht_insert
 │   │       └─ queue_enqueue
 │   ├─ ui_exit
 │   │   └─ vehicle_exit
 │   │       ├─ ht_find
 │   │       ├─ calculate_fee
 │   │       ├─ ht_remove
 │   │       ├─ queue_is_empty / queue_dequeue
 │   │       ├─ stack_push
 │   │       └─ ht_insert
 │   ├─ ui_find
 │   │   ├─ find_vehicle
 │   │   └─ find_in_queue
 │   ├─ ui_revenue / ui_transactions / ui_floor_usage / ui_peak_hours
 │   │   └─ stats_* 系列函数
 │   └─ ui_save / ui_load
 │       └─ save_data / load_data
 ├─ save_data
 └─ ds_destroy
```

整体调用关系说明：主函数负责系统生命周期；UI 层负责接收用户操作；业务层负责入场、出场、查找和计费调度；数据结构层提供栈、队列、哈希表等基础操作；统计和持久化模块负责报表分析与数据保存。
