# 立体多层停车场管理系统三人分工方案

## 一、项目概述

本项目为“数据结构与算法课程实践”的综合性实验，主题为 **立体多层停车场管理系统**。

系统模拟一个多层停车场的日常运行过程，支持车辆入场、车位分配、车辆出场、费用结算、满位排队、实时查找、收入统计、历史交易查看和数据保存加载等功能。

本系统使用 C 语言实现，重点体现数据结构在实际业务中的应用，包括：

- 三维数组：管理多层停车位。
- 栈：管理空闲车位，实现 O(1) 车位分配和释放。
- 队列：管理满位时的等待车辆，实现 FIFO 排队。
- 哈希表：根据车牌快速查找车辆。
- 动态数组：保存历史交易记录。
- 文件读写：实现数据持久化。

## 二、系统核心业务需求

系统需要满足以下业务需求：

| 序号 | 业务需求 | 说明 |
|---|---|---|
| 1 | 车辆快速入场 | 车辆到达入口后，系统自动判断是否有空位，并分配车位 |
| 2 | 车辆快速出场 | 车辆离场时，系统自动查找车辆记录、计算费用并释放车位 |
| 3 | 空位快速分配 | 车位分配和释放尽量达到 O(1) 时间复杂度 |
| 4 | 满位排队 | 停车场无空位时，车辆进入等待队列 |
| 5 | 自动调度 | 有车辆出场后，队首等待车辆自动入场 |
| 6 | 实时查找 | 根据车牌快速查询车辆是否在场或排队 |
| 7 | 自动计费 | 根据停车时长和计费规则自动计算停车费 |
| 8 | 收入统计 | 统计总收入、历史交易、楼层热度和高峰时段 |
| 9 | 数据持久化 | 支持保存和加载停车场数据 |
| 10 | 菜单交互 | 管理员和值班人员可以通过菜单完成操作 |

## 五、核心数据结构设计

### 5.1 停车场模型

本系统设计为 3 层停车场，每层 4 行 5 列：

```text
FLOORS = 3
ROWS   = 4
COLS   = 5
TOTAL_SPOTS = 3 * 4 * 5 = 60
```

每个车位使用三维坐标唯一表示：

```c
/**
 * @brief 车位三维坐标。
 *
 * 该结构体用于唯一定位立体停车场中的一个车位。
 * 程序内部采用从 0 开始的下标，便于直接访问数组；
 * 对用户显示时再统一加 1，例如 floor=0 显示为 1F。
 */
typedef struct {
    int floor;  ///< 楼层编号，范围 0 ~ FLOORS-1，本项目为 0~2
    int row;    ///< 行编号，范围 0 ~ ROWS-1，本项目为 0~3
    int col;    ///< 列编号，范围 0 ~ COLS-1，本项目为 0~4
} SpotLocation;
```

用户看到的位置可以显示为：

```text
第 floor+1 层，第 row+1 行，第 col+1 列
```

### 5.2 全局系统状态

系统使用一个 `ParkingSystem` 结构体保存全部运行状态：

```c
/**
 * @brief 停车场系统全局状态。
 *
 * ParkingSystem 是整个程序最核心的数据结构，相当于系统的“上下文”。
 * pzx、yjk、zxc 三个模块的绝大多数函数都通过 ParkingSystem *ps
 * 访问或修改系统状态。
 *
 * 设计思路：
 * 1. spots 保存真实车位状态；
 * 2. free_spots 保存当前所有空闲车位，便于 O(1) 分配；
 * 3. wait_queue 保存满位时正在等待的车辆；
 * 4. plate_index 根据车牌快速定位在场车辆；
 * 5. history 保存已经完成的出场交易；
 * 6. fee_rule 保存当前计费规则；
 * 7. total_revenue 保存累计收入。
 */
typedef struct {
    /**
     * @brief 停车场全部车位的三维数组。
     *
     * 第一维表示楼层，第二维表示行，第三维表示列。
     * 例如 spots[0][1][2] 表示第 1 层第 2 行第 3 列的车位。
     * 每个 ParkingSpot 中保存该车位的位置、状态和占用车辆车牌。
     */
    ParkingSpot spots[FLOORS][ROWS][COLS];

    /**
     * @brief 车牌到在场车辆记录的哈希索引。
     *
     * key 为车牌号，value 为 VehicleRecord 指针。
     * yjk 在车辆入场成功后插入索引，在车辆出场后删除索引。
     * 该结构用于支持根据车牌 O(1) 平均复杂度查找车辆。
     */
    HashMap plate_index;

    /**
     * @brief 空闲车位栈。
     *
     * 栈中保存当前所有空闲车位的 SpotLocation。
     * 车辆入场时从栈中弹出一个空位；
     * 车辆出场且等待队列为空时，将释放车位重新压入栈。
     */
    FreeSpotStack free_spots;

    /**
     * @brief 等待车辆队列。
     *
     * 当停车场没有空位时，新到车辆进入该队列。
     * 队列采用 FIFO 原则，先到车辆优先获得后续释放的车位。
     */
    WaitingQueue wait_queue;

    /**
     * @brief 历史交易记录动态数组。
     *
     * 每当车辆成功出场并完成结算后，系统追加一条
     * TransactionRecord。统计模块根据该数组显示交易明细、
     * 收入统计和高峰时段分析。
     */
    TransactionRecord *history;

    /**
     * @brief 当前已经保存的历史交易数量。
     *
     * 取值范围为 0 ~ history_capacity。
     * 每成功完成一次车辆出场结算，该值加 1。
     */
    int history_count;

    /**
     * @brief 历史交易数组容量。
     *
     * history 指向的动态数组最多可保存 history_capacity 条记录。
     * 当 history_count 达到容量上限时，应通过 realloc 扩容。
     */
    int history_capacity;

    /**
     * @brief 当前停车场计费规则。
     *
     * 包括免费时长、起步价、每小时费用和每日封顶金额。
     * yjk 的 calculate_fee 函数根据该规则计算出场费用。
     */
    FeeRule fee_rule;

    /**
     * @brief 停车场累计收入。
     *
     * 每次车辆出场结算成功后，yjk 将本次停车费累加到该字段。
     * zxc 的统计模块可直接读取该字段显示总收入。
     */
    double total_revenue;
} ParkingSystem;
```

该结构体是所有模块协作的核心，几乎所有业务函数都以 `ParkingSystem *ps` 作为参数。

字段之间的关系如下：

- `spots` 是车位状态的真实数据源，显示车位图和楼层热度时主要遍历它。
- `free_spots` 是从 `spots` 中抽取出来的“当前空位集合”，用于快速分配。
- `plate_index` 是从在场车辆记录派生出的“车牌索引”，用于快速查找。
- `wait_queue` 保存尚未获得车位的车辆，这些车辆不会出现在 `plate_index` 中。
- `history` 保存已经完成的停车交易，车辆出场后才会写入。
- `fee_rule` 被 yjk 的计费函数依赖。
- `total_revenue` 是由每次出场交易费用累加得到的统计字段。

因此，`ParkingSystem` 不是简单的数据堆放，而是把车位、车辆、队列、索引、交易和计费规则组织到同一个运行上下文中。

## 6.2 pzx 需要定义的数据类型

### 6.2.1 车位状态枚举

```c
typedef enum {
    SPOT_FREE,      // 车位空闲
    SPOT_OCCUPIED   // 车位已被车辆占用
} SpotStatus;
```

注释要求：

- 必须说明枚举值代表什么。
- 不要只写“状态”，要明确“空闲”和“占用”的业务含义。

### 6.2.2 入场结果枚举

```c
typedef enum {
    ENTRY_SUCCESS,  // 入场成功，车辆已分配车位
    PARKING_FULL,   // 停车场已满，车辆进入等待队列
    PLATE_EXISTS    // 车牌重复或非法
} EntryResult;
```

### 6.2.3 出场结果枚举

注意：不能使用 `EXIT_SUCCESS` 作为枚举名，因为 `<stdlib.h>` 中已经定义了 `EXIT_SUCCESS` 宏。

推荐使用：

```c
typedef enum {
    PARK_EXIT_SUCCESS,  // 出场成功
    NOT_FOUND,          // 未找到该车辆
    NOT_PARKED          // 车辆在等待队列中，但尚未停车
} ExitResult;
```

## 6.3 pzx 需要实现的函数

### 6.3.1 系统初始化和销毁

```c
void ds_init(ParkingSystem *ps);
void ds_destroy(ParkingSystem *ps);
```

`ds_init` 需要完成：

1. 初始化所有车位为空闲。
2. 初始化空闲车位栈。
3. 将 60 个车位坐标全部压入空闲车位栈。
4. 初始化哈希表。
5. 初始化等待队列。
6. 初始化历史交易数组，默认容量为 100。
7. 设置默认计费规则。
8. 设置总收入为 0。

`ds_destroy` 需要完成：

1. 释放空闲车位栈动态数组。
2. 清空哈希表节点。
3. 清空等待队列节点。
4. 释放历史交易数组。
5. 将相关指针置为 `NULL`。

### 6.3.2 空闲车位栈操作

```c
void stack_init(FreeSpotStack *s, int capacity);
void stack_destroy(FreeSpotStack *s);
void stack_push(FreeSpotStack *s, SpotLocation loc);
SpotLocation stack_pop(FreeSpotStack *s);
int stack_is_empty(const FreeSpotStack *s);
int stack_size(const FreeSpotStack *s);
```

设计目标：

- 入场分配车位时使用 `stack_pop`。
- 出场释放车位时使用 `stack_push`。
- 栈操作时间复杂度为 O(1)。

### 6.3.3 等待队列操作

```c
void queue_init(WaitingQueue *q);
void queue_clear(WaitingQueue *q);
void queue_enqueue(WaitingQueue *q, const char *plate, time_t t);
int queue_dequeue(WaitingQueue *q, char *out_plate, time_t *out_t);
int queue_is_empty(const WaitingQueue *q);
int queue_count(const WaitingQueue *q);
```

设计目标：

- 停车场满位时，车辆进入队尾。
- 有车位释放时，队首车辆优先进入。
- 队列遵循 FIFO 原则。

### 6.3.4 哈希表操作

```c
unsigned int hash_str(const char *str);
void ht_init(HashMap *ht);
void ht_insert(HashMap *ht, VehicleRecord *rec);
VehicleRecord *ht_find(HashMap *ht, const char *plate);
void ht_remove(HashMap *ht, const char *plate);
void ht_clear(HashMap *ht);
```

设计目标：

- 根据车牌快速查找在场车辆。
- 使用拉链法处理哈希冲突。
- 平均查找、插入、删除复杂度为 O(1)。

## 7.3 yjk 车辆入场逻辑

车辆入场流程：

```text
输入车牌
   ↓
校验车牌
   ↓
检查是否已经在场
   ↓
检查是否已经排队
   ↓
判断是否有空闲车位
   ├── 有空位：分配车位，创建 VehicleRecord，插入哈希表
   └── 无空位：加入等待队列
```

必须处理的情况：

| 情况 | 处理方式 |
|---|---|
| 车牌为空 | 返回 `PLATE_EXISTS` 或扩展为非法车牌 |
| 车牌过长 | 不允许入场 |
| 车牌已在场 | 返回 `PLATE_EXISTS` |
| 车牌已在等待队列 | 返回 `PLATE_EXISTS` |
| 有空车位 | 分配车位并返回 `ENTRY_SUCCESS` |
| 无空车位 | 加入等待队列并返回 `PARKING_FULL` |

## 7.4 yjk 车辆出场逻辑

车辆出场流程：

```text
输入车牌
   ↓
根据车牌查找 VehicleRecord
   ↓
是否在场？
   ├── 不在场：判断是否在等待队列
   │       ├── 在队列：返回 NOT_PARKED
   │       └── 不存在：返回 NOT_FOUND
   ↓
计算停车时长
   ↓
计算停车费用
   ↓
追加历史交易记录
   ↓
更新总收入
   ↓
释放车位
   ↓
删除哈希表索引并释放 VehicleRecord
   ↓
等待队列是否为空？
   ├── 不为空：队首车辆自动占用刚释放车位
   └── 为空：车位压回空闲车位栈
```

核心要求：

- 费用计算必须在释放车辆记录之前完成。
- 历史交易必须在释放车辆记录之前生成。
- 如果等待队列不为空，释放车位应优先分配给队首车辆，而不是压回空闲栈。

## 7.5 yjk 计费逻辑

默认规则：

| 计费项 | 默认值 |
|---|---|
| 免费时长 | 15 分钟 |
| 起步价 | 5 元 |
| 每小时收费 | 4 元 |
| 每日封顶 | 40 元 |

计费公式：

```text
如果停车时长 <= 免费时长：
    费用 = 0
否则：
    超出分钟数 = 停车时长 - 免费时长
    超出小时数 = ceil(超出分钟数 / 60.0)
    费用 = 起步价 + 超出小时数 * 每小时收费
    如果费用超过每日封顶：
        费用 = 每日封顶
```

### 8.2.4 主函数

```c
int main(void);
```

主函数流程：

```text
创建 ParkingSystem
   ↓
调用 ds_init 初始化系统
   ↓
尝试加载历史数据
   ↓
进入菜单循环
   ↓
退出前保存数据
   ↓
调用 ds_destroy 释放内存
```

## 9.3 zxc 调用原则

zxc 在 UI 层不应该直接修改底层结构，例如：

```c
ps->spots[f][r][c].status = SPOT_OCCUPIED;
```

这类操作应交给 yjk 的业务函数处理。

zxc 应通过以下接口完成业务：

```c
vehicle_entry(ps, plate);
vehicle_exit(ps, plate, &fee);
find_vehicle(ps, plate);
find_in_queue(ps, plate);
```

这样可以保证业务规则集中在 yjk，避免 UI 层和业务层逻辑混乱。


## 10.2 源文件注释规范

每个 `.c` 文件开头应包含实现说明：

```c
/**
 * @file parking.c
 * @brief 停车业务逻辑实现。
 *
 * 本文件由 yjk 负责，实现车辆入场、车辆出场、排队调度、

```

## 10.3 结构体注释规范

结构体注释应说明：

1. 该结构体表示什么业务对象。
2. 每个字段的含义。
3. 字段之间的约束关系。

示例：

```c
/**
 * @brief 车位位置坐标。
 *
 * 使用 floor、row、col 三个整数唯一标识停车场中的一个车位。
 * 程序内部从 0 开始编号，显示给用户时通常加 1。
 */
typedef struct {
    int floor;  ///< 楼层编号，范围 0 ~ FLOORS-1
    int row;    ///< 行编号，范围 0 ~ ROWS-1
    int col;    ///< 列编号，范围 0 ~ COLS-1
} SpotLocation;
```

示例：

```c
/**
 * @brief 单个停车位。
 *
 * 空闲时 status 为 SPOT_FREE，plate 为空字符串。
 * 占用时 status 为 SPOT_OCCUPIED，plate 保存占用车辆车牌。
 */
typedef struct {
    SpotLocation loc;   ///< 车位的三维坐标
    SpotStatus status;  ///< 车位当前状态
    char plate[16];     ///< 占用车辆车牌，空闲时为 ""
} ParkingSpot;
```

## 10.4 函数注释规范

每个对外函数必须使用块注释说明。

标准格式：

```c
/**
 * @brief 函数功能一句话描述。
 *
 * 详细说明函数处理流程、业务意义和注意事项。
 *
 * @param 参数名 参数含义。
 * @return 返回值含义。
 *
 * @note 特殊说明，例如调用前置条件、内存释放责任等。
 */
```

示例：

```c
/**
 * @brief 初始化停车场系统。
 *
 * 初始化所有车位为空闲状态，将所有车位坐标压入空闲车位栈，
 * 同时初始化车牌哈希表、等待队列、历史交易数组和默认计费规则。
 *
 * @param ps 停车场系统指针，不能为空。
 *
 * @note 本函数会为 free_spots.spots 和 history 分配动态内存，
 *       使用结束后必须调用 ds_destroy 释放。
 */

```

示例：

```c
/**
 * @brief 车辆入场登记。
 *
 * 系统先检查车牌是否合法以及是否重复；如果存在空闲车位，
 * 则从空闲车位栈中弹出一个车位分配给车辆，并创建在场记录；
 * 如果停车场已满，则将车辆加入等待队列。
 *
 * @param ps 停车场系统指针。
 * @param plate 车辆车牌号，长度不能超过 15。
 * @return ENTRY_SUCCESS 表示入场成功；
 *         PARKING_FULL 表示停车场满，车辆进入等待队列；
 *         PLATE_EXISTS 表示车牌重复或非法。
 */
EntryResult vehicle_entry(ParkingSystem *ps, const char *plate);
```

示例：

```c
/**
 * @brief 车辆出场结算。
 *
 * 根据车牌查找车辆在场记录，计算停车时长和费用，生成历史交易，
 * 更新总收入，释放车位，并在等待队列非空时自动安排队首车辆入场。
 *
 * @param ps 停车场系统指针。
 * @param plate 出场车辆车牌号。
 * @param out_fee 输出参数，用于返回本次停车费用；可为 NULL。
 * @return PARK_EXIT_SUCCESS 表示出场成功；
 *         NOT_FOUND 表示未找到车辆；
 *         NOT_PARKED 表示车辆在等待队列中但尚未停车。
 *
 * @note 本函数会释放对应 VehicleRecord 的动态内存。
 */
ExitResult vehicle_exit(ParkingSystem *ps, const char *plate, double *out_fee);
```

## 10.5 关键逻辑注释规范

关键逻辑需要在代码内部补充简短注释。  
注释重点说明业务原因，而不是重复代码字面意思。

推荐写法：

```c
/* 释放出的车位优先分配给等待队列队首车辆，实现“出一个进一个”。 */
if (!queue_is_empty(&ps->wait_queue)) {
    ...
} else {
    stack_push(&ps->free_spots, released_loc);
}
```

不推荐写法：

```c
/* if 判断队列是否为空 */
if (!queue_is_empty(&ps->wait_queue)) {
    ...
}
```

原因：  
第二种注释只是重复代码，没有解释业务意义。

## 10.6 内存管理注释规范

凡是涉及 `malloc`、`realloc`、`free` 的地方，必须说明内存归属。

示例：

```c
/**
 * @brief 创建车辆在场记录。
 *
 * @note 返回的 VehicleRecord 由业务层持有，
 *       插入哈希表后仍需在车辆出场时由 vehicle_exit 释放。
 */
static VehicleRecord *create_vehicle_record(...);
```

释放时也应注释：

```c
/* ht_remove 只释放哈希表节点，不释放 VehicleRecord，因此这里必须手动释放 rec。 */
ht_remove(&ps->plate_index, plate);
free(rec);
```

## 10.7 文件持久化注释规范

保存和加载数据时必须说明文件中写入了什么，以及为什么某些数据不直接保存。

示例：

```c
/**
 * @brief 保存停车场运行数据到二进制文件。

```

## 十二、错误处理规范

### 12.1 参数检查

所有对外函数应检查关键参数：

```c
if (ps == NULL) {
    return;
}
```

对于有返回值的函数，应返回合理错误值：

```c
if (ps == NULL || plate == NULL) {
    return NOT_FOUND;
}
```

### 12.2 内存分配检查

所有 `malloc` 和 `realloc` 后必须检查：

```c
ptr = malloc(sizeof(Type));
if (ptr == NULL) {
    fprintf(stderr, "memory allocation failed\n");
    exit(EXIT_FAILURE);
}
```

### 12.3 文件打开检查

```c
FILE *fp = fopen(filename, "wb");
if (fp == NULL) {
    printf("无法打开文件 %s\n", filename);
    return 0;
}
```

### 12.4 字符串长度检查

车牌数组长度为 16，因此最多保存 15 个字符和字符串结束符：

```c
if (strlen(plate) == 0 || strlen(plate) >= 16) {
    return PLATE_EXISTS;
}
```

复制时建议使用：

```c
strncpy(dest, src, 15);
dest[15] = '\0';
```

```c
 * @brief 每层停车区域的列数。
 */
#define COLS 5

/**
 * @brief 停车场总车位数。
 *
 * TOTAL_SPOTS = FLOORS * ROWS * COLS。
 * 本系统中总车位数为 60。
 */
#define TOTAL_SPOTS (FLOORS * ROWS * COLS)

/**
 * @brief 车牌哈希表桶数组大小。
 *
 * 使用质数 10007 作为桶数量，可以减少哈希冲突。
 * 当前停车场最多只有 60 个车位，因此该哈希表容量充足。
 */
#define HT_SIZE 10007
```

枚举注释建议：

```c
/**
 * @brief 车位状态枚举。
 *
 * SPOT_FREE 表示车位当前没有车辆占用；
 * SPOT_OCCUPIED 表示车位当前已经分配给某辆车。
 */
typedef enum {
    SPOT_FREE,
    SPOT_OCCUPIED
} SpotStatus;

/**
 * @brief 车辆入场结果枚举。
 *
 * yjk 的 vehicle_entry 函数返回该枚举，zxc 的 UI 层根据返回值
 * 输出不同提示信息。
 */
typedef enum {
    ENTRY_SUCCESS,  ///< 入场成功，车辆已获得车位
    PARKING_FULL,   ///< 停车场已满，车辆已进入等待队列
    PLATE_EXISTS    ///< 车牌重复或输入非法
} EntryResult;

/**
 * @brief 车辆出场结果枚举。
 *
 * 注意不要命名为 EXIT_SUCCESS，因为 C 标准库 stdlib.h 中已经
 * 定义了 EXIT_SUCCESS 宏。
 */
typedef enum {
    PARK_EXIT_SUCCESS, ///< 出场成功
    NOT_FOUND,         ///< 未找到该车牌
    NOT_PARKED         ///< 车辆在等待队列中，但尚未真正停车
} ExitResult;
```

结构体注释建议：

```c
/**
 * @brief 车位三维坐标。
 *
 * 用 floor、row、col 唯一确定停车场中的一个车位。
 * 例如 floor=0,row=1,col=2 表示第 1 层第 2 行第 3 列。
 */
typedef struct {
    int floor;  ///< 楼层编号，范围 0 ~ FLOORS-1
    int row;    ///< 行编号，范围 0 ~ ROWS-1
    int col;    ///< 列编号，范围 0 ~ COLS-1
} SpotLocation;

/**
 * @brief 单个车位对象。
 *
 * 每个车位都有固定的位置 loc、当前状态 status 和占用车辆 plate。
 * 当车位空闲时，status 为 SPOT_FREE，plate 为空字符串。
 * 当车位被占用时，status 为 SPOT_OCCUPIED，plate 保存车牌号。
 */
typedef struct {
    SpotLocation loc;   ///< 车位位置
    SpotStatus status;  ///< 车位状态
    char plate[16];     ///< 占用车辆车牌，最多 15 个字符
} ParkingSpot;

/**
 * @brief 当前在场车辆记录。
 *
 * 该结构体只记录已经真正获得车位的车辆，不记录等待队列中的车辆。
 * 车辆出场时，yjk 会根据该记录计算停车时长和费用。
 */
typedef struct {
    char plate[16];        ///< 车辆车牌号
    SpotLocation loc;      ///< 车辆当前停放位置
    time_t entry_time;     ///< 车辆真正入场停车的时间
} VehicleRecord;

/**
 * @brief 空闲车位栈。
 *
 * 该结构体用于保存当前所有空闲车位的位置。
 * 车辆入场时从栈顶弹出一个空位，实现 O(1) 车位分配；
 * 车辆出场且等待队列为空时，将释放的车位重新压回栈中。
 *
 * 使用栈而不是每次遍历车位数组，是为了避免入场时从 60 个车位中
 * 逐个查找空位，从而提高分配效率。
 */
typedef struct {
    SpotLocation *spots;  ///< 动态数组，存储所有当前空闲车位坐标
    int top;              ///< 栈顶下标，-1 表示栈为空
    int capacity;         ///< 当前栈容量，不足时可扩容
} FreeSpotStack;

/**
 * @brief 等待队列节点。
 *
 * 当停车场没有空闲车位时，新到车辆不会丢弃，而是以节点形式
 * 加入等待队列。由于车辆尚未真正停车，所以节点中只保存车牌号
 * 和进入等待队列的时间，不保存车位坐标。
 */
typedef struct QueueNode {
    char plate[16];             ///< 等待车辆车牌号，最多 15 个字符
    time_t arrival_time;        ///< 车辆进入等待队列的时间
    struct QueueNode *next;     ///< 指向下一个等待车辆节点
} QueueNode;

/**
 * @brief 等待车辆队列。
 *
 * 等待队列采用链式队列结构，遵循先进先出 FIFO 原则。
 * front 指向最早等待的车辆，rear 指向最后加入的车辆。
 * 当有车位释放时，yjk 应优先取出 front 车辆安排入场。
 */
typedef struct {
    QueueNode *front;  ///< 队头指针，指向最早等待的车辆
    QueueNode *rear;   ///< 队尾指针，指向最后加入的车辆
    int count;         ///< 当前等待车辆数量
} WaitingQueue;

/**
 * @brief 哈希表链表节点。
 *
 * 哈希表用于根据车牌号快速查找在场车辆。
 * 由于不同车牌可能计算出相同哈希值，因此每个桶使用链表解决冲突。
 * HTNode 就是桶链表中的一个节点。
 */
typedef struct HTNode {
    char plate[16];          ///< 哈希关键字：车辆车牌号
    VehicleRecord *record;   ///< 指向该车牌对应的在场车辆记录
    struct HTNode *next;     ///< 指向同一哈希桶中的下一个节点
} HTNode;

/**
 * @brief 车牌哈希索引表。
 *
 * HashMap 的 key 是车牌号，value 是 VehicleRecord 指针。
 * 它不直接保存车辆完整信息，而是保存到 VehicleRecord 的引用。
 * yjk 在车辆入场时插入索引，在车辆出场时删除索引。
 */
typedef struct {
    HTNode *buckets[HT_SIZE];  ///< 哈希桶数组，每个桶是一个链表头指针
    int count;                 ///< 当前哈希表中保存的在场车辆数量
} HashMap;

/**
 * @brief 停车计费规则。
 *
 * 该结构体保存当前停车场的收费标准。
 * 系统初始化时设置默认规则，管理员也可以通过 UI 修改规则。
 * 出场结算时，yjk 的 calculate_fee 会读取该结构体计算停车费。
 */
typedef struct {
    double base_fee;       ///< 起步价，例如 5.0 元
    int free_minutes;      ///< 免费停车时长，例如 15 分钟
    double rate_per_hour;  ///< 超过免费时长后每小时收费，例如 4.0 元
    double max_daily;      ///< 每日封顶金额，例如 40.0 元
} FeeRule;

/**
 * @brief 历史交易记录。
 *
 * 每辆车成功出场并完成结算后，系统生成一条历史交易记录。
 * 该记录用于后续收入统计、交易明细查看、楼层热度分析和高峰时段分析。
 */
typedef struct {
    char plate[16];          ///< 出场车辆车牌号
    SpotLocation loc;        ///< 本次停车使用过的车位位置
    time_t entry_time;       ///< 车辆入场时间
    time_t exit_time;        ///< 车辆出场时间
    int duration_minutes;    ///< 停车总时长，单位为分钟
    double fee;              ///< 本次停车产生的费用
} TransactionRecord;

/**
 * @brief 停车场系统全局状态。
 *
 * ParkingSystem 聚合了系统运行需要的全部数据，是 pzx、yjk、zxc
 * 三个模块共同操作的核心对象。
 *
 * 字段关系：
 * - spots 是真实车位状态；
 * - free_spots 是 spots 中空闲车位的快速分配结构；
 * - wait_queue 保存未分配车位的等待车辆；
 * - plate_index 保存已经在场车辆的车牌索引；
 * - history 保存已经完成的出场交易；
 * - fee_rule 决定出场费用如何计算；
 * - total_revenue 保存累计收入。
 */
typedef struct {
    ParkingSpot spots[FLOORS][ROWS][COLS];  ///< 停车场全部车位三维数组
    HashMap plate_index;                    ///< 车牌到在场车辆记录的哈希索引
    FreeSpotStack free_spots;               ///< 当前空闲车位栈
    WaitingQueue wait_queue;                ///< 当前等待入场车辆队列
    TransactionRecord *history;             ///< 历史交易动态数组
    int history_count;                      ///< 当前历史交易记录数量
    int history_capacity;                   ///< 历史交易数组容量
    FeeRule fee_rule;                       ///< 当前停车计费规则
    double total_revenue;                   ///< 系统累计停车收入
} ParkingSystem;
```

### 24.1.2 空闲车位栈接口

```c
/**
 * @brief 初始化空闲车位栈。
 *
 * 为栈分配 capacity 个 SpotLocation 的动态数组空间，并将栈顶 top
 * 初始化为 -1。该栈用于保存当前所有空闲车位坐标。
 *
 * @param s 空闲车位栈指针。
 * @param capacity 初始容量，一般设置为 TOTAL_SPOTS。
 */
void stack_init(FreeSpotStack *s, int capacity);

/**
 * @brief 销毁空闲车位栈。
 *
 * 释放栈中动态分配的 spots 数组，并将指针置为 NULL，
 * 将 top 设置为 -1，capacity 设置为 0。
 *
 * @param s 空闲车位栈指针。
 */
void stack_destroy(FreeSpotStack *s);

/**
 * @brief 将一个空闲车位坐标压入栈。
 *
 * 当车辆出场并且等待队列为空时，释放出的车位需要重新加入空闲车位
 * 集合，此时业务层调用本函数将车位坐标压入栈顶。
 *
 * @param s 空闲车位栈指针。
 * @param loc 待压入的车位坐标。
 *
 * @note 如果当前容量不足，应自动扩容。
 */
void stack_push(FreeSpotStack *s, SpotLocation loc);

/**
 * @brief 从空闲车位栈弹出一个车位坐标。
 *
 * 当车辆入场且存在空闲车位时，业务层调用本函数获取一个可分配车位。
 * 栈顶弹出操作只修改 top，因此时间复杂度为 O(1)。
 *
 * @param s 空闲车位栈指针。
 * @return 栈顶车位坐标。
 *
 * @note 调用前应先使用 stack_is_empty 判断栈是否为空。
 */
SpotLocation stack_pop(FreeSpotStack *s);

/**
 * @brief 判断空闲车位栈是否为空。

 * @return 栈为空返回非 0；否则返回 0。
 */
int stack_is_empty(const FreeSpotStack *s);

/**
 * @brief 获取当前空闲车位数量。

 * @param s 空闲车位栈指针。
 * @return 当前栈中保存的车位坐标数量。

int stack_size(const FreeSpotStack *s);
```

### 24.1.3 等待队列接口

```c
/**
 * @brief 初始化等待队列。
 *
 * 将队头 front、队尾 rear 初始化为 NULL，将 count 初始化为 0。
 *
 * @param q 等待队列指针。
 */
void queue_init(WaitingQueue *q);

/**
 * @brief 清空等待队列。
 *
 * 释放等待队列中的所有 QueueNode 节点，并将队列恢复为空队列状态。
 *
 * @param q 等待队列指针。
 */
void queue_clear(WaitingQueue *q);

/**
 * @brief 将车辆加入等待队列队尾。
 *
 * 当停车场已满时，新到车辆不能立即获得车位，需要加入等待队列。
 * 队列采用 FIFO 原则，先进入队列的车辆先获得释放车位。
 *
 * @param q 等待队列指针。
 * @param plate 车辆车牌号。
 * @param t 车辆进入等待队列的时间。
 */
void queue_enqueue(WaitingQueue *q, const char *plate, time_t t);

/**
 * @brief 从等待队列队头取出一辆车。
 *
 * 当有车辆出场释放车位后，如果等待队列不为空，则调用本函数取出
 * 队首车辆，并由业务层将其安排到释放出的车位上。
 *
 * @param q 等待队列指针。
 * @param out_plate 输出参数，用于保存出队车辆车牌。
 * @param out_t 输出参数，用于保存该车辆进入等待队列的时间。
 * @return 成功出队返回 1；队列为空返回 0。
 */

/**
 * @brief 判断等待队列是否为空。
```

### 24.2.1 停车业务接口

```c
/**
 * @brief 办理车辆入场。
 *
 * 系统首先检查车牌是否合法，以及该车牌是否已经在场或已经排队。
 * 如果存在空闲车位，则分配车位、创建在场车辆记录并插入哈希表；
 * 如果停车场已满，则将车辆加入等待队列。
 *
 * @param ps 停车场系统指针。
 * @param plate 入场车辆车牌号。
 * @return ENTRY_SUCCESS 表示入场成功；
 *         PARKING_FULL 表示停车场已满且车辆进入等待队列；
 *         PLATE_EXISTS 表示车牌非法、已在场或已排队。
 */
EntryResult vehicle_entry(ParkingSystem *ps, const char *plate);

/**
 * @brief 办理车辆出场结算。
 *
 * 根据车牌查找在场车辆记录，计算停车时长和费用，生成历史交易，
 * 更新总收入，释放车位并删除车牌索引。如果等待队列非空，则队首
 * 车辆自动获得刚释放的车位。
 *
 * @param ps 停车场系统指针。
 * @param plate 出场车辆车牌号。
 * @param out_fee 输出参数，用于返回本次停车费用；可以为 NULL。
 * @return PARK_EXIT_SUCCESS 表示出场成功；
 *         NOT_FOUND 表示未找到车辆；
 *         NOT_PARKED 表示车辆在等待队列中但尚未停车。
 */
ExitResult vehicle_exit(ParkingSystem *ps, const char *plate, double *out_fee);

/**
 * @brief 查询在场车辆。
 *
 * 根据车牌号从哈希表中查找车辆当前停车记录。
 *
 * @param ps 停车场系统指针。
 * @param plate 待查询车牌号。
 * @return 找到返回 VehicleRecord 指针；未找到返回 NULL。
 */
VehicleRecord *find_vehicle(ParkingSystem *ps, const char *plate);

/**
 * @brief 查询车辆在等待队列中的位置。
 *
 * 从等待队列队头开始遍历，查找指定车牌号。
 *
 * @param ps 停车场系统指针。
 * @param plate 待查询车牌号。
 * @return 找到返回从 1 开始的排队序号；未找到返回 -1。
 */
int find_in_queue(ParkingSystem *ps, const char *plate);
```
